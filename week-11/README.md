# Week 11: TeamTask Pro — Dashboard and Admin

**Theme:** Build the admin and analytics layer. Ship v1.

You've built authentication, multi-tenant organizations, Stripe payments, and notifications. TeamTask Pro has real features and real functionality. But it's missing the layer that makes it feel like a real product: dashboards that show what's happening, admin tools that let org owners manage their teams, and data export that lets users get their data out.

This week you build that layer. And when you're done, you ship v1. This is the milestone -- a complete, functional SaaS application, live on the internet, built entirely by directing Claude Code.

---

## Learning Objectives

1. Understand what makes a useful dashboard -- which metrics matter and how to present them visually.
2. Understand data visualization concepts -- when to use bar charts, line charts, pie charts, and why.
3. Understand what admins need -- user management, usage statistics, subscription status.
4. Understand aggregation queries conceptually -- counting, summing, grouping data to answer questions.
5. Understand activity logs -- audit trails that record who did what and when.
6. Understand data export -- letting users get their own data out of your application.
7. Ship v1 of TeamTask Pro -- a complete, production-ready SaaS application.

## What You Ship

The **v1 launch of TeamTask Pro** -- with an org-level dashboard showing task analytics and member activity, an admin panel for managing the organization, an activity log, and CSV data export. This is the complete product.

## Time Commitment

~12-15 hours across the week.

## Prerequisites

- **Week 10 completed.** TeamTask Pro has auth, multi-tenant orgs, Stripe payments, and notifications.
- Your TeamTask Pro repo on GitHub with Vercel deployment.
