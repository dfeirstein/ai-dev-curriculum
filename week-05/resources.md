# Week 5 Resources

Reference material for databases, APIs, and validation. Focus on conceptual understanding -- these links help you understand what the tools do, not how to code with them.

---

## PostgreSQL

- [PostgreSQL About Page](https://www.postgresql.org/about/) -- What Postgres is and why it's the most popular open-source relational database. A good conceptual starting point.
- [PostgreSQL Tutorial -- What is PostgreSQL?](https://www.postgresqltutorial.com/postgresql-getting-started/what-is-postgresql/) -- Non-technical overview of Postgres concepts: tables, rows, columns, relations.

## Neon (Hosted Postgres)

- [Neon Documentation](https://neon.tech/docs/introduction) -- Getting started guide. Focus on the "Create a project" and "Connect" sections.
- [Neon Free Tier](https://neon.tech/docs/introduction/plans) -- What you get on the free plan. More than enough for learning projects.

## Drizzle ORM

- [Drizzle ORM Documentation](https://orm.drizzle.team/docs/overview) -- The official overview. The "Why Drizzle?" section explains the philosophy.
- [Drizzle with Neon](https://orm.drizzle.team/docs/get-started/neon-new) -- Specific guide for using Drizzle with Neon Postgres.
- [Drizzle Migrations](https://orm.drizzle.team/docs/migrations) -- How migrations work. Conceptual understanding only -- Claude Code runs these for you.

## Zod

- [Zod Documentation](https://zod.dev/) -- The official docs. The introduction explains what Zod does and why. Browse the "Basic usage" section to understand the concept of schema-based validation.

## Next.js API Routes

- [Next.js Route Handlers](https://nextjs.org/docs/app/building-your-application/routing/route-handlers) -- How API routes work in the App Router. Focus on the concepts, not the code examples.

## REST APIs (Conceptual)

- [REST API Tutorial -- What is REST?](https://restfulapi.net/) -- Conceptual overview of REST as an architectural style. Explains GET, POST, PUT, DELETE, and status codes.
- [HTTP Status Codes](https://developer.mozilla.org/en-US/docs/Web/HTTP/Status) -- Reference for what each status code means. Useful when debugging API issues.

## Environment Variables

- [Next.js Environment Variables](https://nextjs.org/docs/app/building-your-application/configuring/environment-variables) -- How Next.js handles environment variables. Important for understanding the .env.local pattern.
- [Vercel Environment Variables](https://vercel.com/docs/projects/environment-variables) -- How to set environment variables for production deployment.
