# Week 16 Resources

Resources for testing, monitoring, performance, documentation, and shipping. Use these when you want to go deeper on any topic from this week.

---

## Testing

- **[Vitest Documentation](https://vitest.dev/)** -- The official Vitest docs. Reference for understanding test configuration and patterns.
- **[Playwright Documentation](https://playwright.dev/docs/intro)** -- End-to-end testing framework. Understand what E2E tests can do and how they work conceptually.
- **[Testing Trophy (Kent C. Dodds)](https://kentcdodds.com/blog/the-testing-trophy-and-testing-classifications)** -- A practical framework for deciding what to test and how. Focuses on integration tests as the highest-value investment.

---

## Monitoring

- **[Sentry Next.js Guide](https://docs.sentry.io/platforms/javascript/guides/nextjs/)** -- How Sentry integrates with Next.js. Useful reference if you hit configuration issues.
- **[PostHog Next.js Guide](https://posthog.com/docs/libraries/next-js)** -- PostHog integration guide for Next.js. Reference for event tracking patterns.

---

## Performance

- **[Web.dev Performance](https://web.dev/learn/performance)** -- Google's guide to web performance. Conceptual understanding of what makes sites fast.
- **[Lighthouse Documentation](https://developer.chrome.com/docs/lighthouse/)** -- Understanding Lighthouse scores and what they measure.
- **[Next.js Image Optimization](https://nextjs.org/docs/app/building-your-application/optimizing/images)** -- How Next.js handles image optimization. Useful for understanding what Claude Code sets up when you ask for image optimization.
- **[Core Web Vitals](https://web.dev/articles/vitals)** -- The metrics Google uses to measure user experience. LCP, FID, CLS -- what they are and why they matter.

---

## Documentation

- **[Make a README](https://www.makeareadme.com/)** -- A guide to writing good READMEs. Simple, practical, and directly applicable.
- **[Awesome README Examples](https://github.com/matiassingers/awesome-readme)** -- A curated list of excellent READMEs. Browse these for inspiration on how to present your project.

---

## CI/CD

- **[GitHub Actions Documentation](https://docs.github.com/en/actions)** -- The official docs for GitHub Actions. Reference for understanding workflow configuration.
- **[Vercel Git Integration](https://vercel.com/docs/deployments/git)** -- How Vercel automatically deploys from Git pushes. Understand the pipeline you're setting up.

---

## Shipping Mindset

- **[Ship It (Seth Godin)](https://seths.blog/2010/06/fear-of-shipping/)** -- A short essay on why shipping is the hardest and most important part of building anything.
- **[The Cult of Done Manifesto](https://medium.com/@bre/the-cult-of-done-manifesto-724ca1c2ff13)** -- Thirteen principles about finishing things. "Done is the engine of more."

---

## A Note on Quality

This week is about applying everything you've learned about quality -- independently, to your own project. The resources above are references, not tutorials. By now, you know the workflow: describe what you want to Claude Code, verify the output, iterate. The quality playbook is the same whether it's TeamTask Pro or your own project. The skill is knowing when to apply which tool.
