# Project: Portfolio Assembly

**What you're building:** A complete, polished professional portfolio packaging all your work.

**Time estimate:** 10-12 hours across the week

**What you'll need open:** Ghostty, your browser, all your project repos

---

## The Brief

You've built the projects. Now you package them. By the end of this week, every project has a polished README, working live demos, screenshots, and your portfolio site ties it all together with a blog post about your journey.

---

## Phase 1: Audit (Day 1)

### Step 1: Verify All Live Demos

Visit every project's live URL. Check that each one works:

- [ ] **Personal landing page** (Week 1) -- loads, links work
- [ ] **Portfolio site** (Week 7) -- loads, navigation works
- [ ] **TeamTask Pro** (Weeks 8-14) -- loads, auth works, core features work
- [ ] **Independent project** (Weeks 15-16) -- loads, auth works, core features work
- [ ] **Speed build** (Week 18) -- loads, auth works, core features work

If anything is broken, fix it now. A broken demo in your portfolio is worse than no demo at all.

> "Check this project for any runtime errors or broken features. Fix anything that prevents the core functionality from working."

### Step 2: Inventory Your Projects

For each project, note:
- Current state of the README (missing? outdated? complete?)
- Whether screenshots exist
- Whether the live demo link is correct
- Whether the GitHub repo is public
- What needs the most work

---

## Phase 2: Polish READMEs (Days 1-2)

### Step 3: TeamTask Pro README

This is your flagship project. The README should be exceptional.

> "Rewrite the README.md for TeamTask Pro. Make it professional and compelling. Include:
> - A specific description: 'A multi-tenant task management SaaS with team workspaces, role-based access, Stripe billing, email notifications, and analytics dashboards.'
> - Screenshot placeholder (I'll add the actual image)
> - Live demo link: [your URL]
> - Features list (8-10 features, written from the user's perspective)
> - Tech stack (organized by category: Frontend, Backend, Database, Auth, Payments, Testing, Monitoring, CI/CD, Deployment)
> - Getting started instructions
> - Environment variables with descriptions"

### Step 4: Independent Project README

> "Rewrite the README.md for [your project name]. Include a compelling description, features list, tech stack, live demo link, getting started instructions, and environment variable documentation."

### Step 5: Speed Build README

> "Rewrite the README.md for [your speed build]. Note in the description that this was built in 5 days. Include features, tech stack, live demo link, and setup instructions."

### Step 6: Review All READMEs

Read each README as if you've never seen the project before. Ask yourself:
- Do I understand what this does from the first two lines?
- Can I see a screenshot?
- Can I click a live demo link?
- Could I run this locally by following the instructions?

Fix anything that doesn't pass the test.

---

## Phase 3: Screenshots (Day 2-3)

### Step 7: Take Screenshots for Every Project

For each project, capture:
- The main view / dashboard with data
- The core feature in action
- One or two additional interesting views

Tips:
- Use a clean browser window (no bookmarks bar, no extensions)
- Add realistic-looking data if the app looks empty
- Use consistent browser width for all screenshots
- Save screenshots in each project's repository (e.g., `/screenshots/` folder)

### Step 8: Update READMEs with Screenshots

> "Update the README to display the screenshots from the /screenshots folder. Use markdown image syntax and add alt text for accessibility."

---

## Phase 4: Demo Videos (Day 3)

### Step 9: Record Demo Video for TeamTask Pro

Record a 2-minute screen recording walkthrough:

1. Open QuickTime (or your preferred screen recorder)
2. Navigate to TeamTask Pro
3. Walk through: sign up, create a workspace, add tasks, show the dashboard, show notifications
4. Keep it natural -- don't script every click
5. Save the video

### Step 10: Record Demo Video for Your Best Project

Pick your independent project or speed build -- whichever you're most proud of. Record another 2-minute walkthrough.

Upload both videos to YouTube (unlisted is fine) or another hosting service. Add the links to each project's README and your portfolio site.

---

## Phase 5: Update Portfolio Site (Day 3-4)

### Step 11: Add All Projects to Portfolio

Open your portfolio site project and update it with Claude Code:

> "Update my portfolio site to showcase these projects:
>
> 1. TeamTask Pro -- [description, live URL, GitHub URL]
> 2. [Independent project] -- [description, live URL, GitHub URL]
> 3. [Speed build] -- [description, live URL, GitHub URL]
> 4. [Open source contribution or npm package] -- [description, GitHub URL]
>
> For each project, display: a screenshot, the title, a 2-sentence description, the tech stack as tags, and links to the live demo and source code. Order them with TeamTask Pro first."

### Step 12: Update About Page

> "Update the about page. I'm an AI-native builder who has shipped [X] production applications in 20 weeks. I build software by researching problems, planning solutions, and directing AI tools to implement production-quality applications. My stack includes Next.js, TypeScript, Postgres, and the full modern web toolkit."

Customize this with your own voice. Make it sound like you, not a template.

### Step 13: Verify Portfolio Site

Check every link, every screenshot, every project description. Test on mobile. Make sure the portfolio site itself is polished -- this is the first impression.

---

## Phase 6: Blog Post (Day 4-5)

### Step 14: Write Your Blog Post

Choose one topic:

**Option A: "How I Built [Project Name]"**
Walk through one project in detail. The idea, the research, the planning, the build, the challenges, the result.

**Option B: "From Zero to AI-Native Builder in 20 Weeks"**
The full journey. Where you started, what you learned, what surprised you, what's next.

**Option C: "[Specific Topic] I Learned Building a SaaS"**
Deep dive into one concept: multi-tenancy, payment integration, testing, monitoring. What you learned and why it matters.

Write a first draft yourself. Then refine it with Claude Code:

> "Review this blog post. Improve the clarity and flow without changing the voice. Fix any awkward sentences. Suggest a better title if you can think of one."

### Step 15: Publish the Blog Post

Add the post to your portfolio site's blog section:

> "Add this blog post to the blog section of my portfolio site. Format it properly with a title, publication date, and reading time estimate."

Deploy the updated portfolio site.

---

## Phase 7: Final Quality Check (Day 5)

### Step 16: Complete Verification

Go through every item:

- [ ] **All live demos working** -- visit each URL, test core features
- [ ] **All READMEs updated** -- description, features, screenshots, demo links, setup instructions
- [ ] **Screenshots added** -- to each README and the portfolio site
- [ ] **Demo videos recorded** -- for top 2 projects, linked in READMEs
- [ ] **Portfolio site updated** -- all projects, about page, blog post
- [ ] **Blog post published** -- on the portfolio site, well-written and edited
- [ ] **All GitHub repos public** -- anyone can view the source
- [ ] **Mobile responsive** -- portfolio site looks good on phone
- [ ] **No broken links** -- check every link on the portfolio site

### Step 17: Final Deploy

Push all updates. Verify everything one last time on the live sites.
