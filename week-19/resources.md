# Week 19 Resources

Resources for portfolio building, documentation, demo creation, and professional presentation. Use these when you want to go deeper on any topic from this week.

---

## Portfolio Inspiration

- **[Awesome Developer Portfolios](https://github.com/emmabostian/developer-portfolios)** -- A curated list of developer portfolios. Browse these for ideas on layout, content, and presentation.
- **[Brittany Chiang's Portfolio](https://brittanychiang.com/)** -- A widely admired developer portfolio. Notice the clean design, clear project descriptions, and effective use of screenshots.
- **[Max Bock's Portfolio](https://mxb.dev/)** -- Another excellent example. Clean, fast, and content-focused.

---

## README Writing

- **[Make a README](https://www.makeareadme.com/)** -- A practical guide to writing effective READMEs. Covers the essentials without overthinking it.
- **[Awesome README Examples](https://github.com/matiassingers/awesome-readme)** -- A collection of excellent READMEs. Study these for formatting, structure, and content ideas.
- **[GitHub Markdown Guide](https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax)** -- Reference for GitHub-flavored markdown. Useful for formatting your READMEs.

---

## Screenshots and Visuals

- **[CleanShot X](https://cleanshot.com/)** -- A macOS screenshot tool with annotation features. Not free, but excellent if you want professional-looking screenshots.
- **[QuickTime Screen Recording](https://support.apple.com/guide/quicktime-player/record-your-screen-qtp97b08e666/mac)** -- Built into macOS. Free and sufficient for recording demo videos.
- **[OBS Studio](https://obsproject.com/)** -- Free, open source screen recording. More features than QuickTime if you need them.

---

## Writing and Blogging

- **[On Writing Well (William Zinsser)](https://www.goodreads.com/book/show/53343.On_Writing_Well)** -- The classic guide to nonfiction writing. If you want to improve your writing, start here.
- **[Hashnode](https://hashnode.com/)** -- A developer blogging platform. If you want your blog post to reach a wider audience beyond your portfolio site.
- **[Dev.to](https://dev.to/)** -- Another developer blogging platform with a built-in audience. Cross-posting your blog post here can increase visibility.

---

## Professional Presentation

- **[Building in Public (Kevon Cheung)](https://www.buildinpublic.com/)** -- A guide to sharing your work publicly. The philosophy behind writing, sharing, and growing your professional presence.
- **[How to Present a Project (Lara Hogan)](https://larahogan.me/blog/presentation-templates/)** -- Frameworks for presenting technical work effectively.

---

## GitHub Profile

- **[GitHub Profile README](https://docs.github.com/en/account-and-profile/setting-up-and-managing-your-github-profile/customizing-your-profile/managing-your-profile-readme)** -- How to create a README for your GitHub profile. This shows up on your profile page and is a great place for a brief intro and project highlights.
- **[Awesome GitHub Profile READMEs](https://github.com/abhisheknaiidu/awesome-github-profile-readme)** -- Examples of well-crafted GitHub profile READMEs.

---

## A Note on Portfolios

A portfolio is never truly finished. It evolves as you build more. But what you have after this week is a strong foundation -- live projects, documentation, demos, and writing. That's more than enough to start putting yourself out there. Don't wait until it's perfect. Ship it and improve it over time.
