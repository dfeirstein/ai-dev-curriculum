# Week 8 Resources

Reference material for SaaS architecture, multi-tenancy, data modeling, and role-based access control. Focus on concepts and patterns.

---

## SaaS Architecture

- [What is SaaS?](https://www.cloudflare.com/learning/cloud/what-is-saas/) -- Cloudflare's clear explanation of the SaaS model. Good conceptual starting point.
- [Multi-Tenant Architecture](https://www.cloudflare.com/learning/cloud/what-is-multitenancy/) -- How multi-tenancy works and why it matters. Understand the isolation model.
- [The SaaS Starter Stack](https://saasstarterstack.com/) -- A curated list of tools and services commonly used to build SaaS products. Browse for awareness.

## Data Modeling

- [Drizzle ORM Documentation](https://orm.drizzle.team/docs/overview) -- Official Drizzle docs. Focus on the schema definition and relations sections.
- [Drizzle Relations](https://orm.drizzle.team/docs/rqb) -- How to define and query relationships between tables in Drizzle.
- [Database Design Basics](https://www.lucidchart.com/pages/database-diagram/database-design) -- Visual introduction to database relationships (one-to-many, many-to-many). Conceptual overview.

## Authentication and Authorization

- [Better Auth Documentation](https://www.better-auth.com/docs/introduction) -- Official docs. Review the organization/team plugin if available.
- [Better Auth with Next.js](https://www.better-auth.com/docs/integrations/next-js) -- Integration guide specific to your stack.
- [RBAC Explained](https://auth0.com/docs/manage-users/access-control/rbac) -- Auth0's explanation of Role-Based Access Control. Clear conceptual overview that applies regardless of which auth library you use.
- [OAuth 2.0 Simplified](https://aaronparecki.com/oauth-2-simplified/) -- Review from Week 6 if needed. Relevant for GitHub OAuth.

## Project Management UI Patterns

- [Linear](https://linear.app/) -- Sign up for a free account and use it. Notice the UI patterns: project lists, task boards, status columns, keyboard shortcuts. This is your design target.
- [Todoist Design Principles](https://todoist.com/inspiration) -- How Todoist thinks about task management UX.
- [Kanban Board Pattern](https://www.atlassian.com/agile/kanban/boards) -- Atlassian's explanation of the Kanban board pattern you'll build for task management.

## SaaS Landing Pages

- [SaaS Landing Page Examples](https://saaslandingpage.com/) -- Curated collection of real SaaS landing pages. Browse for layout and copy inspiration.
- [Stripe's Landing Page](https://stripe.com/) -- Notice the clear messaging, visual hierarchy, and CTA placement.
- [Linear's Landing Page](https://linear.app/) -- Clean, focused, and effective. A good model for developer tool marketing.

## Monitoring

- [Sentry for Next.js](https://docs.sentry.io/platforms/javascript/guides/nextjs/) -- Review from Week 6. Same integration, bigger project.
- [PostHog for Next.js](https://posthog.com/docs/libraries/next-js) -- Review from Week 6. Focus on tracking the right events for a SaaS product.

## Validation

- [Zod Documentation](https://zod.dev/) -- Review from Week 5. You'll use Zod extensively for form and API validation in a multi-form application.
