# Week 4: The Modern Web Stack

**Theme:** Understand every frontend tool well enough to give Claude Code clear architectural direction.

In Weeks 1-3 you built a personal website, learned the Research-Plan-Execute workflow, and configured Claude Code with CLAUDE.md, custom skills, and hooks. You have a deployed site on Vercel and a solid working relationship with your AI builder.

Now it's time to upgrade your toolkit. This week you'll learn what each piece of the modern frontend stack does -- not how to write code with it, but what it is, why it exists, and how to direct Claude Code to use it well. Then you'll rebuild your personal site using the full stack, turning it from a starter project into something architecturally sound and professionally polished.

---

## Learning Objectives

1. Understand what **Next.js App Router** does and how to direct Claude Code on routing, layouts, and server vs client components.
2. Understand why **TypeScript strict mode** matters as an automated quality gate for AI-generated code.
3. Understand how **Tailwind CSS** works as a design language you can direct Claude Code to apply.
4. Understand what **shadcn/ui** provides and why owning your components matters.
5. Understand the role of **ESLint and Prettier** as automated code quality enforcement.
6. Rebuild your personal site using the full stack with proper architecture.

## What You Ship

A **polished personal website** rebuilt with Next.js App Router, TypeScript strict mode, Tailwind CSS, and shadcn/ui components. Dark mode, responsive design, clean architecture. Deployed to Vercel.

## Time Commitment

~10-12 hours across the week.

## Prerequisites

- **Weeks 1-3 completed.** You have a deployed personal site, understand the Research-Plan-Execute workflow, and have configured CLAUDE.md, custom skills, and hooks.
- Your Week 2 site repo on GitHub with Vercel deployment connected.

## How to Approach This Week

1. **Read guide.md** all the way through. This is conceptual -- you're learning what each tool does and why, so you can give Claude Code better direction.
2. **Work through project.md** with Claude Code open. Direct the rebuild step by step.
3. **Use resources.md** if you want to understand any tool more deeply.

You don't need to understand how these tools work internally. You need to understand what they do, when to use them, and how to tell Claude Code to use them correctly. That's the difference between a camera operator and a director.
