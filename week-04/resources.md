# Week 4 Resources

Reference material for the modern frontend stack. These are for understanding what the tools do -- not for learning to code with them.

---

## Next.js

- [Next.js App Router Documentation](https://nextjs.org/docs/app) -- How the file-based routing system works. Focus on the "Getting Started" and "Routing" sections for the mental model.
- [Next.js Server and Client Components](https://nextjs.org/docs/app/building-your-application/rendering/server-components) -- Explains the server vs client component split. Read the overview, not the implementation details.
- [Next.js Layouts and Templates](https://nextjs.org/docs/app/building-your-application/routing/layouts-and-templates) -- How shared layouts work across pages.

## TypeScript

- [TypeScript for the New Programmer](https://www.typescriptlang.org/docs/handbook/typescript-from-scratch.html) -- A conceptual overview. You don't need to learn the syntax -- just understand what TypeScript does and why.
- [TypeScript in 5 Minutes](https://www.typescriptlang.org/docs/handbook/typescript-in-5-minutes.html) -- The shortest possible introduction. Skim it for the big picture.

## Tailwind CSS

- [Tailwind CSS Documentation](https://tailwindcss.com/docs) -- The full reference. Browse the "Core Concepts" section to understand the approach.
- [Tailwind CSS Responsive Design](https://tailwindcss.com/docs/responsive-design) -- How Tailwind handles mobile, tablet, and desktop layouts.
- [Tailwind CSS Dark Mode](https://tailwindcss.com/docs/dark-mode) -- How dark mode works with Tailwind's class-based approach.

## shadcn/ui

- [shadcn/ui Documentation](https://ui.shadcn.com/) -- The official docs. Browse the "Components" section to see what's available.
- [shadcn/ui Component Gallery](https://ui.shadcn.com/docs/components/accordion) -- Click through the components to see live examples. When you see something you want, just tell Claude Code to use it.
- [shadcn/ui Themes](https://ui.shadcn.com/themes) -- Pre-built color themes you can apply to your project.

## ESLint and Prettier

- [ESLint Getting Started](https://eslint.org/docs/latest/use/getting-started) -- What ESLint does and how it works. You don't configure this yourself -- Claude Code handles it.
- [Prettier Documentation](https://prettier.io/docs/en/) -- What Prettier does. The "Why Prettier?" page is the most useful for understanding the concept.

## Framer Motion (Stretch Goal)

- [Framer Motion Documentation](https://motion.dev/) -- Animation library for React. Browse the examples to see what kinds of animations are possible. Tell Claude Code which ones you want.
