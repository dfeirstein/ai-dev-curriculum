# Week 2: The Claude Code Workflow

**Theme:** From single prompts to a repeatable system for building real software.

Last week you shipped a landing page by talking to Claude Code. This week you learn the *workflow* that professional AI-native builders use every single day: **Research, Plan, Execute.** You will also learn how to teach Claude about your project, how to review what it builds, and how to use Git branches so you always have a safety net.

By the end of the week you will have a polished, multi-page personal website -- built entirely by directing Claude Code -- with a clean Git history and live deployment.

---

## Learning Objectives

1. Master the **Research --> Plan --> Execute** workflow and apply it to every feature you build.
2. Create and maintain a **CLAUDE.md** file that gives Claude persistent context about your project.
3. **Review agent output** using quality gates (build, lint, type-check, visual inspection) instead of reading code line-by-line.
4. Use **Git branches and pull requests** to keep your main branch safe while you experiment.
5. Write **better prompts** -- specific, constrained, and iterative -- so Claude builds what you actually want.
6. **Recover from mistakes** using error messages, Git reverts, and fresh sessions.

## What You Ship

A **4-5 page personal website** built entirely through Claude Code direction:

- Home (upgraded from Week 1)
- About
- Projects
- Blog (placeholder)
- Contact (with a form)

All pages share a responsive navigation bar, follow a consistent design, and are deployed live on Vercel.

## Time Commitment

~10-12 hours across the week.

## Prerequisites

- **Week 1 completed.** You have a deployed landing page, a working Claude Code installation, and you are comfortable giving Claude basic prompts.
- A GitHub account with your Week 1 repo.
- Vercel connected to that repo for automatic deployments.
