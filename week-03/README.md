# Week 3: Skills, Hooks, and Power Tools

**Theme:** Turn Claude Code from a general-purpose assistant into your personal AI development team.

You already know how to direct Claude Code to build things. You know the Research, Plan, Execute workflow. You have a CLAUDE.md that gives Claude context about your project. This week you learn how to **customize Claude Code itself** -- adding new capabilities, automating quality checks, and connecting it to external tools.

By the end of the week, your development environment will feel like a well-oiled machine. You give a command, and the right things happen automatically.

---

## Learning Objectives

1. Understand the **three customization layers** of Claude Code: skills, hooks, and MCP servers.
2. **Install and create custom skills** that teach Claude new workflows specific to your project.
3. **Configure hooks** that automate quality checks -- linting, type-checking, and building -- without you remembering to run them.
4. Understand what **MCP servers** are and how they extend Claude Code's reach to external tools.
5. Organize and maintain the **`.claude/` directory** as the home for all your customizations.
6. Develop the **power user mindset** -- investing in setup that compounds across every future project.

## What You Ship

Your Week 2 personal website, now backed by a **professional-grade development environment**:

- A comprehensive CLAUDE.md with detailed conventions and quality standards
- 2+ custom skills (e.g., /new-page, /check, /deploy)
- Hooks that auto-lint after edits and type-check before commits
- A new "Uses" page added entirely through the custom skill workflow
- Everything passing quality gates and deployed live

## Time Commitment

~10-12 hours across the week.

## Prerequisites

- **Week 2 completed.** You have a deployed multi-page personal website, you know the Research --> Plan --> Execute workflow, and you have a working CLAUDE.md.
- Your Week 2 repo on GitHub with Vercel auto-deployment.
- Comfort directing Claude Code to build features and reviewing the output.
