# Week 15 Resources

Resources for scoping, planning, and building an independent project. Use these when you want to go deeper on any topic from this week.

---

## MVP and Product Thinking

- **[How to Plan an MVP (Y Combinator)](https://www.ycombinator.com/library/4Q-a-minimum-viable-product-is-not-a-product-it-s-a-process)** -- YC's perspective on what an MVP actually is. Focus on the process, not the artifact.
- **[The Lean Startup: MVP Concept](https://theleanstartup.com/principles)** -- The original framework for building minimum viable products. Worth understanding even if you never start a startup.
- **[Shape Up: Shaping (Basecamp)](https://basecamp.com/shapeup/1.1-chapter-02)** -- How Basecamp thinks about scoping projects. The concept of "appetite" -- deciding how much time something is worth -- is directly relevant to your two-week build.

---

## Competitive Research

- **[Product Hunt](https://www.producthunt.com/)** -- Browse what people are launching. Great for seeing what exists in your project's space and how people describe their tools.
- **[AlternativeTo](https://alternativeto.net/)** -- Search for any app and see alternatives. Useful for finding competitors you didn't know about.
- **[IndieHackers](https://www.indiehackers.com/)** -- Solo builders sharing what they've built and how. Good for inspiration and realistic expectations about independent projects.

---

## Architecture and Planning

- **[Drizzle ORM Documentation](https://orm.drizzle.team/docs/overview)** -- Reference for designing your database schema. You don't write the code -- Claude Code does -- but understanding the concepts helps you review its work.
- **[Better Auth Documentation](https://www.better-auth.com/docs/introduction)** -- Reference for auth patterns. Useful when deciding what auth features your project needs.
- **[Next.js App Router Documentation](https://nextjs.org/docs/app)** -- Reference for understanding page structure and routing. Helps you plan your page layout before building.

---

## Building and Deploying

- **[Vercel Deployment Documentation](https://vercel.com/docs/deployments/overview)** -- Reference for deployment. You've done this before, but it's useful for troubleshooting if anything is different with your new project.
- **[shadcn/ui Components](https://ui.shadcn.com/)** -- Browse available components. Knowing what's available helps you make better UI decisions when directing Claude Code.
- **[Tailwind CSS Documentation](https://tailwindcss.com/docs)** -- Reference for understanding the styling system. Again, you don't write CSS, but conceptual understanding helps you give better direction.

---

## Project-Specific Resources

Depending on what you're building, these might be relevant:

- **[Stripe Documentation](https://docs.stripe.com/)** -- If your project involves payments.
- **[Resend Documentation](https://resend.com/docs)** -- If your project sends emails.
- **[Uploadthing Documentation](https://docs.uploadthing.com/)** -- If your project handles file uploads.
- **[OpenAI API Documentation](https://platform.openai.com/docs)** -- If your project uses AI features.
- **[Google Maps API](https://developers.google.com/maps/documentation)** -- If your project involves location or mapping.

---

## A Note on Research

This week, research isn't optional. It's the foundation of everything you build. The best projects come from understanding the problem space deeply before writing a single prompt. Spend the time. Look at competitors. Talk to potential users. Map out your features.

The research phase is what separates a random project from one that actually solves a problem.
