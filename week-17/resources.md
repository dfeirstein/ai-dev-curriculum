# Week 17 Resources

Resources for advanced AI-native patterns, open source contribution, and npm publishing. Use these when you want to go deeper on any topic from this week.

---

## Advanced Prompting

- **[Claude Code Best Practices](https://docs.anthropic.com/en/docs/claude-code/best-practices)** -- Revisit this with fresh eyes. Now that you have 16 weeks of experience, the advanced sections will make more sense.
- **[Prompt Engineering Guide (Anthropic)](https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/overview)** -- Anthropic's guide to effective prompting. The section on chain-of-thought and step-by-step reasoning is particularly relevant this week.

---

## Open Source

- **[How to Contribute to Open Source (GitHub Guide)](https://opensource.guide/how-to-contribute/)** -- The canonical guide to open source contribution. Covers everything from finding projects to submitting PRs.
- **[First Contributions](https://github.com/firstcontributions/first-contributions)** -- A practice repository for making your first open source contribution. Good warm-up if you've never submitted a PR to someone else's project.
- **[Good First Issues](https://goodfirstissues.com/)** -- A curated list of beginner-friendly issues across popular open source projects.
- **[Up For Grabs](https://up-for-grabs.net/)** -- Another aggregator of projects with issues specifically marked for new contributors.

---

## npm Publishing

- **[npm: Creating and Publishing a Package](https://docs.npmjs.com/creating-and-publishing-unscoped-public-packages)** -- The official guide to publishing packages on npm.
- **[Modern npm Package (Sindre Sorhus)](https://github.com/sindresorhus/np)** -- A tool that helps with the npm publishing process. Understanding what it automates helps you understand what's involved.
- **[How to Write an Open Source JavaScript Library (egghead)](https://egghead.io/courses/how-to-write-an-open-source-javascript-library)** -- A course covering the full lifecycle of creating and maintaining an npm package.

---

## Code Review

- **[Google Engineering Practices: Code Review](https://google.github.io/eng-practices/review/)** -- Google's guide to code review. Even though you're using AI for review, understanding what experienced reviewers look for helps you ask better questions.
- **[Conventional Comments](https://conventionalcomments.org/)** -- A system for structuring code review feedback. Useful framework for how you give feedback to Claude Code.

---

## Understanding Codebases

- **[How to Read Other People's Code (freeCodeCamp)](https://www.freecodecamp.org/news/how-to-read-other-peoples-code/)** -- Strategies for navigating unfamiliar code. The mental models here complement the Claude Code exploration patterns from this week's guide.

## A Note on This Week

This week marks a shift. For the first time, you're not just building your own projects -- you're engaging with the broader developer ecosystem. Whether you contribute to open source or publish a package, you're moving from consumer to contributor. That's a meaningful transition. The tools you've built with are tools that someone else contributed. Now you're part of that cycle.
