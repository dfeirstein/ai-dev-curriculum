# Week 7: Project 1 -- Personal Portfolio + Blog

**Theme:** Direct Claude Code to build a complete, polished portfolio site from scratch.

You've spent six weeks learning the tools and concepts. You know the full stack. You've built an authenticated bookmark manager. Now it's time to build something that's entirely yours -- a personal portfolio and blog that showcases who you are and what you're building.

This is your first independent project. You'll use the full research, plan, execute workflow. You'll make design decisions, content decisions, and technical decisions -- then direct Claude Code to bring them to life.

By the end of the week, you'll have a live portfolio site with a blog, deployed on Vercel, optionally on your own custom domain.

---

## Learning Objectives

1. Run a **complete research phase** independently -- study what makes great portfolios work.
2. Make **content strategy decisions** -- what to include, what to write about, how to present yourself.
3. Understand **SEO fundamentals** conceptually -- metadata, Open Graph, sitemaps -- and direct Claude Code to implement them.
4. Understand **MDX** as a content format -- markdown that can include interactive components.
5. Direct Claude Code through a **full build** using everything learned in Weeks 1-6.
6. Adopt a **building in public** mindset -- sharing your work and writing about your learning.

## What You Ship

A **personal portfolio and blog** with project showcases, an about page, blog posts written in MDX, SEO metadata, and a polished design. Deployed to Vercel, optionally on a custom domain.

## Time Commitment

~12-15 hours across the week.

## Prerequisites

- **Weeks 1-6 completed.** You have the full stack knowledge and a working dev environment.
- A GitHub account and Vercel account.
- Ideas for what to write about (your learning journey is a great starting point).
- Optional: a custom domain name (you can buy one from Namecheap, Google Domains, or Cloudflare for ~$10/year).

## How to Approach This Week

1. **Read guide.md** thoroughly. This week is about creative and strategic decisions as much as technical ones.
2. **Do real research** before building. Look at portfolios you admire. Take notes on what works.
3. **Work through project.md** using the research, plan, execute workflow.
4. **Write at least one real blog post.** Your learning journey is interesting. Share it.

This is the first project that's truly yours. The bookmark manager was a guided exercise. This portfolio represents you. Take the time to make it something you're proud of.
