# Week 1 Resources

A curated set of resources to deepen your understanding of the tools and concepts from this week. You don't need to read all of these -- use them when you want to learn more about something specific.

---

## Claude Code

- **[Claude Code Overview](https://docs.anthropic.com/en/docs/claude-code/overview)** -- What Claude Code is, how it works, and what it can do. Start here if you want to understand the tool at a deeper level.
- **[Claude Code Getting Started](https://docs.anthropic.com/en/docs/claude-code/getting-started)** -- The official getting-started guide. Covers installation, first commands, and basic usage.
- **[Claude Code Best Practices](https://docs.anthropic.com/en/docs/claude-code/best-practices)** -- Tips for writing effective prompts and getting better results. Worth skimming even in week 1.

---

## Ghostty (Terminal)

- **[Ghostty Website](https://ghostty.org/)** -- The official site for the Ghostty terminal. Includes documentation on configuration and features.
- **[Terminal Basics (MDN)](https://developer.mozilla.org/en-US/docs/Learn_web_development/Getting_started/Environment_setup/Command_line)** -- A beginner-friendly explanation of what a terminal is and how to use one. If you've never used a terminal before, this is a good primer.

---

## Zed (Editor)

- **[Zed Website](https://zed.dev/)** -- The official site for the Zed editor. Fast, modern, and designed for AI-assisted development.
- **[Zed Documentation](https://zed.dev/docs)** -- Covers setup, keybindings, and features. Browse this if you want to get more comfortable navigating files in Zed.

---

## Git (Version Control)

These resources explain Git **conceptually** -- what it does and why, not command-by-command tutorials.

- **[Git: The Simple Guide](https://rogerdudler.github.io/git-guide/)** -- A no-nonsense visual overview of Git. Good for building a mental model of what's happening when you commit and push.
- **[Atlassian: What Is Git](https://www.atlassian.com/git/tutorials/what-is-git)** -- A clear, conceptual explanation of version control and why it matters. Skips the deep technical details.
- **[GitHub: About Repositories](https://docs.github.com/en/repositories/creating-and-managing-repositories/about-repositories)** -- Understand what a repository is and how GitHub organizes code.

---

## GitHub CLI

- **[GitHub CLI Documentation](https://cli.github.com/manual/)** -- The official reference for `gh`. You won't need to memorize this -- Claude Code handles the commands -- but it's useful to know what's possible.

---

## Vercel (Deployment)

- **[Vercel Documentation](https://vercel.com/docs)** -- The official docs. The "Getting Started" section is most relevant to you right now.
- **[Vercel + Next.js Guide](https://vercel.com/docs/frameworks/nextjs)** -- Specific guidance on deploying Next.js projects to Vercel, which is exactly what you're doing this week.

---

## Next.js (Framework)

You don't need to learn Next.js in detail. These resources help you understand **what it is** and **why we're using it**.

- **[Next.js: What Is Next.js](https://nextjs.org/learn/react-foundations/what-is-react)** -- An approachable introduction to React and Next.js. Focuses on concepts rather than code.
- **[Next.js Documentation](https://nextjs.org/docs)** -- The full docs. You won't need most of this now, but the "Introduction" and "Getting Started" pages are worth a look.
- **[Vercel: Why Next.js](https://vercel.com/solutions/nextjs)** -- A brief explanation of why Next.js is a popular choice for web development.

---

## Tailwind CSS (Styling)

- **[Tailwind CSS: What Is Tailwind](https://tailwindcss.com/docs/utility-first)** -- A quick explanation of the utility-first approach to styling. You don't need to write Tailwind yourself -- Claude Code handles that -- but understanding the philosophy helps you give better feedback.

---

## General Mindset

- **[AI-Native Development (Anthropic Blog)](https://www.anthropic.com/news/claude-code-best-practices)** -- How Claude Code fits into modern software development workflows. Aligns directly with the philosophy of this curriculum.

---

## A Note on These Resources

You'll notice we're not linking to coding tutorials, syntax references, or "learn to code in 30 days" courses. That's intentional. Your job isn't to write code. Your job is to understand the tools, the concepts, and the workflow well enough to direct an AI to build what you envision.

If you find yourself deep in a tutorial trying to memorize syntax, step back. That's not your lane. Your lane is understanding what things are, what they do, and how to tell if they're working. The AI handles the rest.
